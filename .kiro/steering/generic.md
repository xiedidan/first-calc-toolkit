## CLI 工具执行

### 分页器处理
- CLI 工具（如 psql、less、more）默认启用分页器会阻塞命令返回
- 必须禁用分页器：psql 使用 `-P pager=off`，其他工具查阅文档
- 症状：命令执行但无返回，控制台显示 "-- More --" 或类似提示

### 执行结果验证
- 用户可能先于 AI 看到命令输出
- 若用户确认命令成功但 AI 未收到返回，考虑分页器或其他阻塞因素
- 使用绝对路径执行未在 PATH 中的工具

## 跨平台脚本开发

### PowerShell 脚本规范
- 压缩工具使用前先删除目标文件：`Remove-Item -Force -ErrorAction SilentlyContinue`
- 7z 命令使用 `& 7z` 调用，避免参数解析错误
- 检查命令退出码：`$LASTEXITCODE`
- 隐藏详细输出：`| Out-Null`

### Bash 脚本规范
- 使用 `set -e` 遇错即停
- 检查命令可用性：`command -v tool &> /dev/null`
- 提供配置变量在脚本开头，支持环境变量覆盖
- 自动检测失败时提供清晰的手动操作指南

### 离线部署脚本原则
- 禁止自动下载或 pull 任何资源
- 优先使用本地工具，次选本地容器/镜像
- 提供多种回退方案，确保可用性
- 版本兼容性问题优先选择通用格式（如 SQL vs 二进制）
